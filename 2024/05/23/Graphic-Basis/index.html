1. 渲染管线

  • 渲染管线是一个流水线过程。在给定虚拟相机，三维物体，光源，纹理等场景要素的前提一下生成一张二维图像。

  • 按照逻辑可划分为：应用程序阶段，几何阶段，光栅化阶段，像素处理阶段

    每个阶段会把前一个阶段的输出作为该阶段的输入

    通常应用程序阶段在 CPU 上完成，是完全可编程的。剩余阶段在 GPU 上完成，是部分可编程的。

1. 应用程序阶段

这一阶段的主要任务是完成一些预处理工作和准备渲染数据以及提交数据到 GPU。

  • 预处理工作

    由开发者具体决定。可能包括可见性判断 (剔除 / Culling)，动画，物理，碰撞检测等

    • 这一阶段的剔除通常是粗粒度的 (是逐对象的，而不是逐多边形)
    • An In-Depth Look at Real-Time Rendering

  • 准备渲染数据

    • 摄像机，模型 (材质，纹理，Shader)，光源等
    • 设置渲染状态: 如定义模型材质，光源以及使用的 Shader’
    • 其中最重要的是顶点数据 (顶点坐标，纹理坐标，法线，颜色等)

  • 提交渲染数据

    • 把数据加载到显存中
    • 调用 Draw Call
      • Drawcall 是渲染时采用的单一处理命令，即 CPU 对图形绘制接口的调用，GPU 接受命令
      • Drawcall 过多可能影响性能，尽量把小的 Drawcall 合并到一个大的 Drawcall 中是优化的一种方法，称之为批处理

2. 几何阶段

  • 几何阶段接收来自应用程序阶段的顶点数据，生成光栅化阶段所需的图元数据
  • 图元结构决定了以哪种基本的几何表示来进行渲染。一般有：三角形，四边形，多边形，线段，点。通常使用三角形

1. 顶点着色器

主要用来对 CPU 提交的顶点数据进行坐标变换和顶点着色。

  • 逐顶点执行，本身不创建和销毁任何顶点，且无法得到顶点与顶点之间的关系

    顶点相互独立使得可以利用 GPU 进行高效的并行化处理

  • 坐标变换

    将顶点数据中的顶点坐标从局部空间转换到齐次裁剪空间

    • 模型变换

      将顶点坐标从各自的局部坐标系转换到统一的世界坐标系下

      在变换法线数据时需要注意，如果存在非均匀缩放变换则要使用矩阵的逆的转置来变换法线

    • 观察变换

      将顶点坐标从世界坐标系转换到观察坐标系。

      观察坐标系由虚拟相机的位置与观察方向决定

    • 投影变换

      将顶点坐标从观察坐标系转换到裁剪空间的坐标

      • 视锥体

        视锥体可以通过上下左右远近六个平面定义。摄像机的可见范围就是这些平面框定的空间范围

        透视投影和正交投影的视锥体不同，前者近平面比远平面小，是一个四棱台；后者远近平面大小一致，是一个长方体

      • 裁剪空间

        为了方便计算成像，会将可视的范围空间映射成立方体。这个立方体就是裁剪空间，其长宽高的范围均为 [-w, w]。

      • 投影变换

        使用投影矩阵进行投影变换后， w 分量保留的是顶点观察坐标中 z 坐标的信息，即到摄像机的远近。保留此信息是为了实现透视投影近大远小的效果

  • 顶点着色

    在顶点着色器中计算的着色是逐顶点的。对于一个三角面，会在顶点着色器中计算三个顶点的着色信息，然后在光栅化阶段通过插值得到三角形内部各个片段的光照。

    该方法效果并不理想，因为各片段的着色信息可能非线性变换的，通过插值不能准确反映。此方法亦称为高洛德着色。

2. 曲面细分着色器

曲面细分着色器 (Tessellation Shader) 是渲染管线的一个可选阶段。利用镶嵌化技术对三角面进行细分，以此来增加三角形面数。

组成：

  • 外壳着色器 (Hull Shader)
  • 镶嵌器 (Tessellator)
  • 域着色器 (Domain Shader)

不使用高模而使用曲面细分着色器的原因：

  • 基于 GPU 可实现动态 LOD 技术，在远处渲染低模，近处使用曲面细分着色器增加细节
  • 节省内存，只保存低模信息

3. 几何着色器

  • 几何着色器 (Geometry Shader) 是渲染管线的一个可选阶段，位于曲面细分可光栅化之间
  • 以完整的图元作为输入，例如，以三角形的三个顶点作为输入，然后输出对应的图元
  • 可以创建或销毁几何图元，亦可根据输入图元类型扩展为其他更多类型的图元

4. 图元装配

图元装配 (Primitive Assembly) 将顶点数据组装成指定的图元。它会进行裁剪和背面剔除等相关优化，以减少进入光栅化的图元数量。

  • 裁剪(clipping)

    • 裁剪在透视投影之后，透视除法之前进行
    • 完全位于裁剪空间 (见上文) 之内的图元被保留，完全位于裁剪空间之外的图元被舍弃，部分位于裁剪空间内的图元则会被裁剪掉空间外的部分，并视情况生成新的顶点

  • 背面剔除 (Back-Face Culling)

    剔除背对摄像机的图元，利用三角形顶点点序来确定正面和背面。通常认为 ccw(couter-clockwise) 是正面

    渲染透明或者半透明物体时，使用背面剔除会出现问题。因为此时可以看到物体的背面。

  • 透视除法 (Perspective Division)

    顶点坐标经顶点着色器转换成了齐次裁剪坐标，其中 w 分量保留了 z 坐标信息。此时除以 w 分量，得到了归一化的设备坐标 (Normalized Device Coordinates)，亦称 NDC 坐标 (通常范围在 [-1, 1]之间)。此时的空间被称作标准视体 (Canonical View Volume, CVV)。

  • 屏幕映射 (Sreen Mapping)

    屏幕映射将 NDC 坐标转换成窗口。即将 NDC 坐标的 xy 从 [-1, 1] 映射到 [0, w] 和 [0, h]。w 和 h 分别代表屏幕的宽度和高度。

    屏幕映射不会对 z 坐标作任何处理，此信息在之后用以深度测试。

3. 光栅化

  • 接收来自几何阶段的图元数据，并将其离散为片元
  • 离散化过程是利用三角图元所覆盖的片元来表示该图元
  • 片元只是像素的候选者，只有通过了后续的测试，才会成为最终显示的像素

1. 三角形组装

三角形组装 (Triangle Setup) 会对顶点数据进行插值，计算出每个图元的边界信息。通过三角形组装，我们得到的是若干三角形及其边界。

2. 三角形遍历

  • 三角形遍历(Triangle Traversal) 是寻找三角形所覆盖的所有像素的过程
  • 对三角形所覆盖的若干像素，使用三角形的顶点信息进行插值，就得到了当前三角形在该像素对应的片元数据
  • 片元不等同于像素，在屏幕的同一像素位置，有可能对应多个三角形的不同片元
  • 三角形遍历完成之后，就会生成并输出一个片元序列

4. 像素处理

  • 接收来自光栅化阶段的片元序列，并最终为屏幕每个像素计算着色

1. 片元着色器

  • 片元着色器接收上一阶段生成的片元，并逐片元计算着色
  • 纹理，光照，阴影等一般都在此处计算
  • 并行化，片元之间互不影响

2. 测试和混合

  • 决定每个片元的可见性，包括深度测试，模板测试，Alpha 测试等
  • 如果片元通过了所有测试，它还需要考虑合并
    • 帧缓存中存储了渲染的像素信息
    • 渲染当前帧时是完全覆盖上一帧结果还是进行其他处理是合并需要考虑的问题

上述流程完成，屏幕就能成功显示出帧缓存中的颜色值。
