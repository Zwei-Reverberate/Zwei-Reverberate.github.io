    C++ Reflect

    反射概述

    • 反射（Reflection）是一种让程序自我检查和操作自身结构的能力
    • 反射主要用于检查类、方法、属性等，并进行动态调用和访问
    • 通常用于实现动态类型检查、依赖注入、序列化、测试框架等
    • C++ 暂未提供内建的反射机制，但可以通过一些手段实现类似功能
    • 反射可以分为运行时反射和编译期反射

    运行时反射

    • 运行时反射在程序执行过程中动态地获取类型信息并操作对象
    • 它依赖于运行时的元数据和动态查找，通常会带来一定的性能开销

    RTTI (运行时类型识别)

    C++ 提供的RTTI功能，包括 typeid 操作符和 dynamic_cast 操作符，可以用于运行时类型识别，但其功能有限。

    自定义类型注册和工厂模式

    通过自定义类型注册系统和工厂模式，可以动态创建对象

    • 工厂类负责注册类和创建对象

      • CreateFunc 是一个函数对象类型，表示一个返回 Base* 的无参函数
      • registerClass 函数将类名和创建对象的函数注册到静态注册表中
      • createInstance 函数根据类名查找创建函数并创建对象
      • getRegistry 函数返回一个静态的注册表，该注册表是一个 std::unordered_map，将类名映射到创建函数

    • 使用宏定义进行类注册

      为了方便类的注册，我们使用了一个宏 REGISTER_CLASS

      这个宏定义了一个静态的布尔变量，并在变量定义时执行一个 lambda 表达式

      lambda表达式中调用Factory::registerClass函数，将类名和对应的创建函数注册到工厂中

    • 使用模板代替宏

      可以使用模板来替代宏进行类注册，以提高代码的类型安全性和可读性

    编译时反射

    • 编译期反射（Compile-time Reflection）是在编译阶段收集和利用类型信息的能力
    • 编译期反射的一个显著优势是它可以在不影响运行时性能的情况下进行类型检查和操作

    序列化和反序列化

    • 序列化（Serialization）是将对象的状态转换为可以存储或传输的格式的过程
    • 反序列化（Deserialization）则是将存储或传输的格式还原为对象的过程
    • 主要用于网络通信、持久化存储和数据交换
