    1. 定义模板

    • 模板声明：

      尽量使用 typename 而非 class

    • 模板实例化

      • 在使用模板的地方，编译器会根据模板参数生成具体类型的代码

      • 这个过程发生在编译期。运行期执行生成的具体代码，与普通的非模板代码执行没有区别

      • 模板实例化可能导致编译速度变慢

        • 每个源文件 (编译单元) 在编译时都会独立实例化模板。如果多个源文件使用同一个模板，并且模板的实现被包含在这些源文件中，那么每个编译单元都会实例化该模板

        • 可以通过显式实例化优化这一问题

          • 显式实例化用以避免重复实例化，确保模板实例化只在一个编译单元中发生

          • 一般来说，模板的实现通常放在头文件中。而在源文件中进行显式实例化

    3. 类模板特化

    类模板特化（template specialization）允许为特定的模板参数提供专门的实现。

    类模板特化有两种主要形式：完全特化和部分特化

    • 完全特化（Full Specialization）

      完全特化是针对某个特定类型参数的特化。需要提供完整的类模板特化定义

      适用于类模板和函数模板

    • 部分特化（Partial Specialization）

      部分特化允许为模板参数的一部分提供特化实现，适用于类模板

      部分特化可以处理模板参数的某些特定组合，而不是所有组合

    4. 非类型模板参数

    对于函数模板和类模板，模板参数并不局限于类型，普通值也可以作为模板参数。

    • 整型或枚举类型的常量
    • 指向对象或函数的指针

    • 折叠表达式

      折叠表达式是 C++17 引入的一个特性，用于简化参数包的展开。它允许对参数包中的所有元素应用某个操作符，而不需要显式的递归调用

      • 折叠表达式的语法如下：

        1. 一元左折叠 (unary left fold): ( ... op pack )
        2. 一元右折叠 (unary right fold): ( pack op ... )
        3. 二元左折叠 (binary left fold): ( init op ... op pack )
        4. 二元右折叠 (binary right fold): ( pack op ... op init )
        • op 是二元操作符，如 +, *, &&, || 等。
        • pack 是参数包。
        • init 是初始值，用于二元折叠表达式

      • 一元左折叠

        一元左折叠 ( ... op pack ) 将操作符 op 应用于参数包中的所有元素，从左到右。相当于 ((a op b) op c) op d

      • 一元右折叠

        一元右折叠 ( pack op ... ) 将操作符 op 应用于参数包中的所有元素，从右到左。相当于 a op (b op (c op d))

      • 二元左折叠

        二元左折叠 ( init op ... op pack ) 将初始值 init 和参数包中的所有元素，从左到右依次应用操作符 op。相当于 (((init op a) op b) op c) op d

      • 二元右折叠

        二元右折叠 ( pack op ... op init ) 将初始值 init 和参数包中的所有元素，从右到左依次应用操作符 op。相当于 a op (b op (c op (d op init)))
