    1. 内存布局

    • 主流操作系统的内存模型（x86_64-linux-gnu）

      当一个程序运行时，操作系统为它分配一个进程地址空间，这个空间通常分为几个不同的段，每个段有特定的用途和特点。主要的段包括：

      1. 代码段（.text segment）

        • 存储内容：代码段存储程序的执行代码，即编译后的机器指令
      • 只读：通常是只读的，以防止程序意外或恶意地修改指令
        • 共享：多个进程可以共享相同的代码段，减少内存开销

      1. 数据段（Data Segment）

        数据段分为两部分：

        • 已初始化数据段 （.data segment）

          • 存储内容：存储已初始化的全局变量和静态变量

        • 读写权限：一般是可读写的

        • 未初始化数据段（.bss segment）

          • 存储内容：存储未初始化的全局变量和静态变量

        • 初始化为零：在程序启动时，这些变量被自动初始化为零

          • 读写权限：一般是可读写的

      2. 堆（Heap）

        • 存储内容：堆用于动态分配的内存，在运行时通过 malloc、new 等函数进行分配，通过 free、delete 等函数进行释放

      • 管理方式：堆的内存分配和释放由程序员显式管理

        • 内存布局：通常从低地址向高地址增长

      1. 栈（Stack）

        • 存储内容：栈用于管理函数调用，包括局部变量、函数参数和返回地址

      • 管理方式：由编译器自动管理，具有后进先出（LIFO）特性

        • 内存布局：通常从高地址向低地址增长

      1. 常量区（Text Segment/Read-Only Data Segment）

        • 存储内容：常量区存储只读数据，如字符串字面量和 const 变量

      • 只读：通常是只读的，以防止修改常量数据

    • C++ 的内存模型

      • 每个变量之间都隔着不可访问的真空地带

      • 变量的起始地址未知，真空地带的宽度未知

        所谓未知，指的是由编译器实现细节决定，用户无权干涉，也无需干涉

    • 操作系统内存模型和 C++ 内存模型并不是同一回事

    2. C++ 对象模型

    static 成员变量和成员函数：static 成员变量存储在数据段，成员函数存储在代码段，二者不在对象的内存布局中

    • 对象组成部分

      在 C++ 中，对象通常由以下几个部分组成：

      • 成员变量：对象所包含的数据
      • 成员函数（方法）：对象所能执行的操作
      • 虚表指针（vptr）：用于实现动态多态性的指针（仅在有虚函数的类中存在）
      • 基类子对象：如果类是继承自某个基类，则对象中包含基类的子对象

    • 成员变量的布局

      数据成员在对象内存中的布局由编译器决定，但通常遵循以下规则：

      • 按声明顺序：编译器通常按声明顺序将数据成员排列在内存中
      • 对齐：编译器可能会插入填充字节（padding）以满足对齐要求

      考虑以下类定义：

      Base 类对象布局：baseInt：一个 int 成员

      Derived 类对象布局：

      • 继承自 Base 类，因此包含 Base 类的子对象
      • 追加 derivedDouble 和 derivedChar 成员
      • 编译器可能会插入填充字节以满足对齐要求

    • 成员函数布局

      • 非虚函数

        对于非虚函数，编译器会将它们的代码存储在代码段（.text 段）。非虚函数没有特殊的对象级别的布局需求，它们不存储在对象的内存布局中

      • 虚函数和虚表

        当类包含虚函数时，每个对象将包含一个指向虚表（vtable）的指针（vptr）

        虚表是一个函数指针数组，包含类的虚函数地址，同一类的实例共享同一张虚表

        每个包含虚函数的类的对象都有一个指向虚表的指针，称为虚表指针（vptr）。vptr 通常是对象内存布局中的第一个成员（编译器实现细节，实际位置可能不同）

      考虑以下类定义：

      Base 类对象布局：包含 vptr 和 baseInt

      Derived 类对象布局：

      • 包含 vptr、baseInt、derivedDouble 和 derivedChar
      • vptr 指向 Derived 类的虚表，覆盖了 func1 的实现

      每个基类子对象将按声明顺序排列在内存中，每个基类的 vptr 也将包含在对象中

      • 内存访问：成员变量的布局和对齐影响内存访问的效率
      • 动态多态性：虚函数调用涉及通过 vptr 查找 vtable，略有开销
      • 内存占用：多重继承和虚表增加了对象的内存占用

    2. 值类别

    表达式（Expression）

    在 C++ 中，表达式是任何可以计算出一个值的构造。表达式包括常量、变量、函数调用、操作符运算、成员访问表达式、类型转换表达式等。例如：

    • 常量表达式：42
    • 变量表达式：x
    • 函数调用表达式：foo()
    • 操作符表达式：x + y
    • 成员访问表达式：obj.member
    • 类型转换表达式：static_cast<int>(3.14)

    值类别描述了表达式在程序中的角色和用法。

    “类型”描述了表达式所代表的数据的性质和行为。二者是从不同维度对于表达式的描述。

    1. 分类

    值类别主要分为以下几类：

    • 左值（Lvalue）：左值表示持久存储的对象，即在表达式结束后仍然存在的对象

      • 可以出现在赋值操作符的左侧（即可以被赋值）
      • 可以取地址
      • 具有名字（如变量）
      • 例子：变量名、解引用指针、数组元素、字符串字面值

    • 右值（rvalue）：右值表示临时对象或字面值，它们在表达式结束后会被销毁

      • 不能出现在赋值操作符的左侧（即不可被赋值）

      • 通常没有名字，存在于表达式计算过程中

      • 不能取地址

      • 例子：字面值、临时对象、返回的非引用值

      • 右值分为纯右值和将亡值

        • 纯右值

          纯右值是一种右值，主要用于产生临时对象或计算结果

          • 在表达式计算过程中产生
          • 例子：字面值（如整数、浮点数）、返回的非引用类型的临时对象

        • 将亡值

          将亡值是一种右值，表示即将被销毁的对象，通常用于移动语义

          • 临时对象，即将被销毁的对象
          • 主要出现在右值引用的上下文中
          • 例子：返回右值引用的函数结果、通过 std::move 转换的对象

    移动构造函数在以下情况下特别有用：

    • 返回局部对象：函数返回局部对象时，移动构造函数可以避免不必要的拷贝

    • 临时对象：在使用临时对象时，可以通过移动构造函数避免资源的复制

    • 容器操作：在标准容器中，如 std::vector 或 std::map，移动构造函数可以显著提高性能

    3. 函数对象

    • 函数对象（Function Object），也称为仿函数（Functor），是一个实现了 operator() 的类或结构体实例
    • 通过重载 operator()，一个对象可以像函数一样被调用。这种机制使得函数对象比普通函数具有更多的灵活性和状态性，可以携带和维护状态信息
    • C++ 提供了多种方式来处理函数对象和函数调用，包括 std::function、std::bind 以及 lambda 表达式

    1. std::function

    std::function 是一个通用的、多态的函数包装器。它能够存储、复制和调用任意可调用目标（函数、lambda 表达式、绑定表达式或其它函数对象），其类型签名为 std::function<R(Args...)>，其中 R 是返回类型，Args... 是参数类型列表

    2. std::bind

    std::bind 是一个函数模板，用于创建函数对象，将某个函数或函数对象的一些参数绑定到特定的值上。它的返回值是一个新的函数对象，可以像普通函数一样调用。

    在 std::bind 中，std::placeholders::_1, std::placeholders::_2 等占位符表示调用时传入的参数。占位符的索引表示参数在绑定表达式中的位置。

    3. Lambda 表达式

    Lambda 表达式是用于定义匿名函数的简洁方式。它们可以在定义处捕获外部变量，并且可以像普通函数一样调用：

    • capture：捕获列表，指定哪些外部变量在 lambda 表达式中可用
    • parameters：参数列表
    • return_type：返回类型（可选，编译器通常能自动推断）
    • body：函数体

    Lambda 表达式的类型是一个编译器生成的匿名类类型：

    • 当定义一个 lambda 表达式时，编译器实际上创建了一个匿名类，这个类包含了：
      • 捕获列表中的变量（作为成员变量）
      • 函数调用运算符 operator() 的实现
    • std::function：可以用于包装 lambda 表达式，提供统一的函数调用接口

    捕获列表指定了 lambda 表达式如何捕获外部变量：

    • [&]：按引用捕获所有外部变量
    • [=]：按值捕获所有外部变量
    • [x]：按值捕获变量 x
    • [&x]：按引用捕获变量 x

    4. C++ 元组

    1. std::tie

    std::tie 是一个函数模板，用于将多个变量绑定到一个元组中。它的主要作用是创建一个元组，并将传入的变量按顺序绑定到元组的成员中。通常情况下，std::tie 用于结构化绑定（structured binding），即将一个或多个变量绑定到一个元组或其他结构中。

    示例用法：

    • std::tie(a, b, c) 将变量 a、b 和 c 绑定到一个元组中，并返回该元组
    • 修改元组中的元素会直接反映到原始变量中，因为 std::tie 返回的是对原始变量的引用

    5. 多态

    所谓多态，就是同一个函数名具有多种状态，或者说一个接口具有不同的行为；C++的多态分为编译时多态和运行时多态，编译时多态也称为为静态联编，通过重载和模板来实现，运行时多态称为动态联编，通过继承和虚函数来实现

    • 派生、构造与析构

      • 当创建一个派生类对象时，会先调用基类的构造函数，然后调用派生类的构造函数。

        这是因为在构造派生类对象之前，必须先初始化它所继承的基类部分。这保证了派生类在其构造函数开始执行之前，基类部分已经被正确初始化

      • 当销毁一个派生类对象时，会先调用派生类的析构函数，然后调用基类的析构函数

        这是因为在销毁派生类对象之前，必须先销毁派生类部分，然后再销毁基类部分。这保证了在派生类析构函数执行期间，派生类的资源可以被正确释放

    • 构造、析构与虚函数

      • 在基类的构造函数函数和析构函数中调用虚函数不会表现出多态行为，而是调用基类的版本

        在派生类的构造函数和析构函数中调用虚函数可以表现出多态，调用派生类版本

      • 构造函数不能是虚函数：对象构造时，虚表尚未建立

        析构函数可以是虚函数，且当释放一个指向派生类的基类指针时，需要定义子类重写虚析构函数，否则可能会造成析构不完全

    • 虚表

      当类包含虚函数时，每个对象将包含一个指向虚表（vtable）的指针（vptr）

      虚表是一个函数指针数组，包含类的虚函数地址，同一类的实例共享同一张虚表

      每个包含虚函数的类的对象都有一个指向虚表的指针，称为虚表指针（vptr）。vptr 通常是对象内存布局中的第一个成员（编译器实现细节，实际位置可能不同

    • 多继承

      • 多继承允许一个类从多个基类继承成员和方法

      • 菱形继承

        菱形继承结构中，一个类通过多个路径继承自同一个基类

        这种结构会导致基类成员被多次继承，从而产生二义性问题

        示例：

        两个父类会对公共基类的数据和方法产生一份拷贝，因此对于子类来说读写一个公共基类的数据或调用一个方法时，不知道是哪一个父类的数据和方法。这就是二义性

      • 虚继承

        菱形继承可通过虚继承解决 (继承公共基类时用virtual修饰)。

        虚继承的目的是让某个类做出声明，承诺愿意共享它的基类。其中，这个被共享的基类就称为虚基类（Virtual Base Class）。不论虚基类在继承体系中出现了多少次，在派生类中都只包含一份虚基类的成员。

        在此例中 Base 是被共享的基类。

        在使用虚继承时，构造函数的调用顺序有所变化。虚基类的构造函数会在最初被调用，并且只会被调用一次

    6. 内存

    1. 分配和释放

    • C

      在 C 中，内存分配和释放主要使用以下两个函数：

      malloc：分配指定字节数的内存，返回指向已分配内存的指针。内存中的内容未初始化

      free：释放由 malloc、calloc 或 realloc 分配的内存，不会调用析构函数

    • C++

      在 C++ 中，内存分配和释放使用操作符：

      new：分配内存并调用构造函数进行初始化

      delete：释放由 new 分配的内存，并调用析构函数

    区别：

    • 初始化

      malloc 不初始化分配的内存，new 会调用构造函数初始化对象

    • 类型安全

      malloc 返回 void*，需要显式转换为具体类型的指针。new 直接返回指定类型的指针，无需类型转换

    2. 对齐

    • 存储单位

      1 字节（B） = 8 比特（bit）

      千字节（KB） = 1024 字节（B）

      1 兆字节（MB） = 1024 千字节（KB）

      1 千兆字节（GB） = 1024 兆字节（MB）

      1 太字节（TB） = 1024 千兆字节（GB）

      1 拍字节（PB） = 1024 太字节（TB）

    • 定义

      内存对齐（Memory Alignment）是指将数据存储在内存中的地址按照特定的边界排列

    • 对齐原则

      1. 结构体整体字节数是最大成员字节数的整数倍

      2. 结构体每个成员地址的偏移量要是该成员字节数的整数倍

        看如下示例：

        最大成员字节数为 8 字节，对齐值为 8 字节，整体大小为 16 字节。c 占 1 字节，但 i 的偏移量要是 4 的整数倍，所以在 c 之后需要填充 3 个字节的 padding，使 i 的起始地址为 4 的整数倍。

      3. 结构体的对齐值为最大成员的对齐值

        因为我手动指定了 Stm 的对齐值为 16 字节，在 MyStruct 它又是最大成员，所以 MyStruct 的对齐值也为 16 字节

      • 性能优化：CPU 以固定大小的块为单位（如 4 字节）访问内存。如果数据是对齐的，就可以减少访问次数。声明数据时按照对齐的原则也可以节省内存
      • 兼容要求：某些架构强制要求内存对齐（如传递给 Vulkan Shader 的 Uniform 数据）

    • 良好的对齐方式：

      能够对齐的数据依次声明，小成员可进行组合。不要把小成员声明在字节对齐的数据之间

      • alignas 用于显式指定类型的对齐方式，即手动指定类型的对齐值（只能指定 2 的幂次方）
      • alignof：获取类型的对齐要求，即获取类型的对齐值

    7. const

    const 关键字在 C++ 中用于定义常量，具有不可变性。

    可以用于修饰变量、指针、函数参数、成员函数等

    • 修饰变量

      将变量声明为常量，意味着该变量的值在初始化后不能被修改

    • 修饰指针

      const 可以用于修饰指针的不同部分，包括指针自身和指针所指向的对象

      • 指向常量的指针：指针所指向的对象是常量，不能通过该指针修改对象的值（底层 const）

      • 常量指针（const pointer）：指针自身是常量，不能改变指针的指向（顶层 const）

      • 指向常量的常量指针（const pointer to const）：指针自身和指针所指向的对象都是常量

    • 修饰函数参数

      将函数参数声明为 const，可以防止在函数内部修改传入的参数

    • 修饰成员函数

      将成员函数声明为 const，可以保证成员函数不会修改对象的成员变量。const 成员函数只能调用其他 const 成员函数

    • 修饰成员变量

      类中的成员变量可以声明为 const，但必须在构造函数的初始化列表中进行初始化

    顶层 const 和底层 const：

    • 顶层 const（top-level const）：指的是常量本身，如 const int x
    • 底层 const（low-level const）：指的是指针或引用所指向的对象是常量，如 const int* p

    8. 类型转换

    1. 隐式类型转换

    隐式类型转换是编译器在必要时自动进行的类型转换，通常发生在以下几种情况中：

    • 赋值操作

      当把一个类型的值赋给另一种类型的变量时，如果两种类型兼容，编译器会自动进行类型转换

    • 算术运算

      在算术运算中，不同类型的操作数会被自动转换为同一种类型，以便进行计算

      算术转换规则：

      • 整型提升：小于 int 的整型类型（如 char 和 short）会被提升为 int
      • 浮点数转换：如果操作数中有浮点数，整型会被转换为浮点数。float 会被提升为 double

    • 比较运算

      在比较运算中，不同类型的操作数也会被自动转换为同一种类型，以便进行比较

    • 函数调用

      当传递参数给函数时，如果函数形参和实参的类型不一致，编译器会自动进行类型转换

    9. 智能指针

    Resource Acquisition Is Initialization (RAII) 的简称，是 C++ 中一种管理、避免泄露的惯用法。利用的就是 C++ 构造的对象最终会被销毁的原则。

    RAII 的做法是使用一个对象，在其构造时获取对应的资源，在对象生命周期内控制对资源的访问，使之始终保持有效，最后在对象析构的时候，释放构造时获取的资源。这样做的好处是不用开发人员显式地去释放资源，避免因考虑不周而造成的资源泄露。

    智能指针是一个对象，利用 C++ 的 RAII 机制来实现。当智能指针释放的时候，智能指针的析构函数就会释放其管理的内存资源，无需开发人员手动释放已经分配的内存空间。

    10. 翻译单元

    1. 定义

    在 C++ 中，翻译单元（translation unit）是编译的基本单位：

    • 由一个源文件及其直接或间接包含的所有头文件组成
    • 是源代码文件及其所有包含文件（包括头文件）展开后的结果
    • 每个翻译单元在编译过程中单独处理，生成一个目标文件（object file），这些目标文件最终由链接器组合在一起，形成可执行文件或库
    • 在编译过程中，编译器只关心当前翻译单元的内容，而不考虑其他翻译单元

    2. 区分声明和定义

    • 声明（Declaration）

      告诉编译器某个变量、函数、类或其他实体的名称和类型，但不分配内存或生成代码

    • 定义（Definition）

      定义不仅包含声明，还包含实际创建实体所需的所有信息

      对于变量，定义分配内存。对于函数，定义包含函数的实现。对于类，定义包含成员变量和成员函数的实现

    3. 单一定义规则

    C++ 中的单一定义规则（One Definition Rule, ODR）是确保程序的正确性和一致性的重要规则：

    • 单一定义规则：在整个程序中，每个变量、函数、类、枚举和模板的定义只能有一个。例如，全局变量在整个程序中只能定义一次，而声明可以出现多次
    • 多次声明：在同一个作用域内，同一个变量或函数可以有多个声明，但必须有且只有一个定义

    4. 符号链接类型

    在 C++ 中，符号（函数和变量）的链接类型决定了符号在编译单元（translation unit）之间的可见性

    链接类型：

    1. 内部链接（Internal Linkage）

      • 具有内部链接的符号只能在声明它的编译单元内访问，不能被其他编译单元访问
      • 通常用 static 关键字修饰
      • 头文件中的内部链接和源文件中的内部链接
        • 当在头文件中声明具有内部链接的符号时，每个包含该头文件的翻译单元都会有一个独立的符号副本
        • 当在源文件中声明具有内部链接的符号时，这些符号仅在该源文件内可见

    2. 外部链接（External Linkage）

      • 具有外部链接的符号可以在多个编译单元中访问

      • 默认情况下，非静态的全局变量和函数是外部链接

      • extern 让多个翻译单元共享同一个声明

        inline 让多个翻译单元共享同一个定义

    3. 无链接（No Linkage）

      • 无链接的符号只能在声明它的块（block scope）内访问，不能在块之外访问
      • 局部变量和函数参数通常具是无链接

    static 关键字用于在全局作用域中声明具有内部链接的符号

    extern 关键字用于声明具有外部链接的符号，但不定义它

    const 和 consexpr 变量默认具有内部链接，如果需要外部链接，需要加 extern 关键字

    12. 内联函数

    C++ 中的内联函数（inline function）是一种优化函数调用的机制。

    内联函数通过将函数体直接插入到调用点处来避免函数调用的开销，如参数传递，栈帧管理等。

    栈帧管理（Stack Frame Management）是函数调用过程中管理函数调用信息的一种机制。每次函数调用时，都会在栈（stack）中创建一个新的栈帧（stack frame）来存储函数的局部变量、参数、返回地址等信息。当函数执行完毕时，相应的栈帧会被销毁。

    1. 适用场景

    • 函数体非常小，函数调用的开销比函数体执行的时间还长
    • 函数频繁调用，减少函数调用开销可以显著提升性能

    2. 优缺点

    优点：

    • 减少函数调用开销：内联函数通过将函数体直接插入到调用点处，避免了函数调用的额外开销
    • 提高性能：对于小函数和频繁调用的函数，内联可以显著提高性能
    • 优化机会：编译器可以更好地优化内联函数，因为它可以看到完整的函数实现
    • 内联函数可以看作是宏的安全替代品：宏在预处理阶段进行简单的文本替换，而内联函数在编译阶段进行更复杂的语义检查，避免了宏的一些常见问题，如意外的副作用和难以调试。

    缺点：

    • 代码膨胀：如果内联函数的函数体较大或被多次调用，代码膨胀（code bloat）可能会导致程序体积增大
    • 内联限制：编译器可能会拒绝内联过于复杂的函数（如递归函数）

    3. inline

    inline 只是表示内联建议，尽管程序员可以使用 inline 关键字建议编译器内联某个函数，但最终是否内联由编译器决定

    inline 现在真正的作用是让多个翻译单元共享同一个定义，突破单一定义规则

    13. 窄化转换

    窄化转换（narrowing conversion）是指将一种较宽范围的数据类型转换为较窄范围的数据类型。这种转换有可能导致数据的损失或者不精确的结果。例如，将一个浮点数转换为整数，或者将较大的整数类型转换为较小的整数类型，都会属于窄化转换。

    在 C++ 中，窄化转换可能会引发以下几种问题：

    1. 精度丢失：从浮点数到整数的转换时，浮点数的小数部分会被丢弃

    2. 溢出：当较大的整数类型被转换为较小的整数类型时，如果数值超出了目标类型的表示范围，会导致数据溢出

    3. 负值转换：将无符号类型转换为有符号类型时，若值超出目标类型的范围，结果可能是不可预测的

    使用列表初始化可以防止无意的窄化转换，所以更推荐使用列表初始化。

    14. 杂项

    1. explicit

    explicit 关键字用于防止隐式类型转换，从而避免不期望的类型转换导致的错误。它主要用在类的构造函数前，使得该构造函数不能用于隐式转换和复制初始化

    2. constexpr

    constexpr 关键字用于指定一个表达式或函数可以在编译时求值

    它允许编译器在编译阶段进行优化，从而避免运行时的计算开销

    • constexpr 修饰变量

      constexpr 变量必须在定义时初始化，并且其值必须是编译时常量

    • constexpr 函数

      constexpr 函数是能够在编译时求值的函数

      在 C++11 中 constexpr 函数只能是单个 return 语句，但在 C++14 及以后只要能由编译器推导出一个返回值即可

      constexpr 函数所有参数和返回值必须是字面值类型
