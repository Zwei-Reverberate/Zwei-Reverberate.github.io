    Algorithm Notes

    1. 表、栈和队列

    抽象数据类型 (abstract data type, ADT) 是带有一组操作的一些对象的集合

    1. 表

    • 形如 A0,A1,A2,...AnA_0, A_1, A_2, ... A_nA0​,A1​,A2​,...An​ 的序列称为表，n = 0 时为空表。

    • 数组实现

      • 内存静态分配，连续
      • 下标访问常数时间
      • 插入删除线性时间，可能需要移动大量元素且可能涉及内存再分配问题

    • 链表实现

      • 由一系列不必在内存中相连的结点组成。每个结点包含表元素和指向后继结点的链 (next)

      • 一般地，如果知道发生变化的位置，插入删除不需要移动大量元素，只需要对结点链接进行固定的几个改变

      • STL 中的 vector 和 list

        • vector 和 list 是 STL 中关于线性表的实现，vector 基于动态数组实现，list 基于双向链表实现

        • vector

          • 优点是在常数时间内索引

          • 缺点是插入删除代价较大，除非这些操作发生在 vector 的末尾

        • list

          • 优点是发生变化的位置已知的话，插入删除的代价是常数时间、

          • 缺点是不易索引

    2. 栈

    • 栈 (stack) 模型
      • 栈是限制插入和删除只能在表末端执行的表，表末端亦称栈顶 (top)
      • 对栈的基本操作是入栈 (push) 和出栈 (pop)，相当于限制在表末端的插入和删除
    • 栈本质是线性表，所以任何实现表的方法都能实现栈

    3. 队列

    • 队列 (queue) 模型
      • 插入在一端进行，删除则在另一端进行
      • 入队 (enqueue): 在表末端 (队尾) 插入一个元素
      • 出队 (dequeue): 在表头 (队头) 删除一个元素

    2. 树

    1. 基本概念

    • 树是若干结点的集合
      • 若为空集则是一棵空树
      • 若非空，则树由根结点 r 以及 n 棵子树组成。这些子树的根结点都被来自根 r 的一条有向边所连接
      • r 是每一棵子树的父结点，每一棵子树的根结点是 r 的子结点
      • 若非空树，则树有 n 个结点和 n-1 条边
    • 没有子结点的结点被称为叶结点
    • 具有相同父节点的结点互为兄弟结点
    • 对于任意结点 nin_ini​，nin_ini​ 的深度等于从根 r 到 nin_ini​ 的唯一路径的长 (连接 r 和 nin_ini​ 边的条数)
      • 根节点 r 的深度为 0
    • 对于任意结点 nin_ini​，nin_ini​ 的高度为它到所有叶结点最长路径的长
      • 所有叶结点的高度为 0
      • 一棵树的高度等于它根的高度

    1. 树的实现

    表示一般树的一个可行方法是对于每个结点都记录第一个子结点和下一个兄弟结点。

    这样对于任意结点，和它直接相连的子节点都能找到。

    2. 树的遍历

    前序遍历：对结点的处理工作在诸子节点之前

    后续遍历：对结点的处理工作在诸子节点之后

    2. 二叉树

    • 二叉树 (binary tree) 是一棵每个结点都不能有多于两个子节点的树
    • 一般而言，二叉树的深度要比结点数 N 小得多

    1. 二叉查找树

    • 对于每个结点 X，其左子树中的所有结点的值都小于 X 的值，其右子树中的所有结点的值都大于 X 的值

    • 二叉查找树 (binary search tree) 平均深度为 O(log⁡N)O(\log{N})O(logN)

    • 一个可用的二叉查找树的实现

      • 数据成员是指向根节点的指针 root
      • contains: 如果树 T 中存在值为 X 的结点，则返回 true，否则返回 false
      • findMin 和 findMax: 这里分别使用了递归和非递归两种实现
      • insert: 使用递归实现，插入一个结点到适当的位置
      • remove
        • 如果要删除的结点是叶节点，可以立即被删除
        • 如果要删除的结点有一个子节点，则可以先将其父节点的链指向其子节点，然后再删除其本身
        • 如果要删除的结点有两个子结点，则首先可以找到其右子树中的最小结点 M。然后以 M 的值代替当前结点的值，最后删除 M。因为 M 是右子树中的最小结点，所以不可能再有左结点，对 M 的删除就回到了前面已述及的两种情况
        • 如果删除的次数不多，通常使用的策略是懒惰删除：当一个元素要被删除时，它仍留在树中，只是做了一个被删除的记号
      • makeEmpty: 将树置为空树，析构函数调用

    2. AVL 树

    • 一般的二叉查找树可能存在某些结点深度过深的问题，这会提高查找的时间成本

    • AVL (Adelson-Velskii and Landis) 是带有平衡条件的二叉查找树

      对于 AVL 树的每个结点，其左子树喝右子树高度最多差1

    • 除了可能的插入外，所有的树操作都可以在以 O(log⁡N)O(\log{N})O(logN) 时间执行

    • 对于插入操作来说，插入一个结点可能破坏 AVL 树的特性。此时恢复 AVL 树的简单修正称之为旋转

      • 把需要重新平衡的结点称为 α\alphaα，α\alphaα 是不满足 AVL 条件的最小树根节点

      • 由于任意结点最多只有两个儿子，因此高度不平衡时，α\alphaα 的两棵子树的高差为2。这种不平衡可能出现在下面四种情况中：

        1. 对 α\alphaα 的左儿子的左子树进行一次插入
        2. 对 α\alphaα 的左儿子的右子树进行一次插入
        3. 对 α\alphaα 的右儿子的左子树进行一次插入
        4. 对 α\alphaα 的右儿子的右子树进行一次插入

        其中 1、4 都发生在外侧，可以归结为一种情况，2、3 归结为一种情况。前者可以通过单旋转处理，后者可以通过双旋转处理。

      • 单旋转

        • 旋转中心：α\alphaα 相应的子结点 (1 为左儿子，4 为右儿子)
        • 旋转方向：1 右旋，即顺时针旋转；4 左旋，即逆时针旋转
        • 旋转操作可简单想象为将旋转中心按旋转方向向上替换自己的父节点

      • 双旋转

        • 旋转中心：α\alphaα 相应的孙子结点 (2 为左儿子的右儿子，3 为右儿子的左儿子)
        • 旋转方向：2 先左旋后右旋，3 先右旋后左旋

    3. B 树

    B 树是一种自平衡的树，能够保持数据有序。用 M 表示一棵 B 树的阶数，即该树中每个结点最多拥有 M 个子结点。

    B 树的键是存储在每个结点种用于排序和检索的值，并在结点中升序排列。例如假设结点第一个键为 x，第二个键的值为 y。那么它第一个子节点所有的键都小于 x，第二个结点所有的键都在 x 和 y 之间。

    一棵 M 阶的 B 树满足的性质如下：

    • 每个结点最多有 M 个子结点。除叶结点和根结点外，每个结点最少有 ⌈M2⌉\lceil \frac{M}{2} \rceil⌈2M​⌉ 个结点。如果根节点不是叶结点，那么它至少有两个结点
    • 每个结点最多有 M - 1 个键。刚好用来分隔子节点的最大数量 M 个
    • 除叶结点和根结点外，每个结点最少有 ⌈M2⌉\lceil \frac{M}{2} \rceil⌈2M​⌉ - 1 个键
    • 所有叶结点都在同一层

    4. 红黑树

    红黑树（Red-Black Tree）是一种平衡二叉查找树（BST），它在插入和删除操作后，通过少量的颜色变换和旋转操作来保持树的平衡，从而保证了查找、插入和删除的时间复杂度都是 O(log n)。红黑树的性质如下：

    1. 每个节点是红色或黑色
    2. 根节点是黑色
    3. 每个叶子节点是黑色
    4. 如果一个节点是红色的，则它的两个子节点都是黑色的（即不存在两个连续的红色节点）
    5. 对每个节点，从该节点到其所有后代叶子节点的路径上，包含相同数目的黑色节点

    一个可用的实现：

    • 颜色和节点定义：

      • Color 枚举定义了节点的颜色（红色或黑色）
      • Node 结构体定义了红黑树的节点，包括数据、颜色、左右子节点和父节点指针

    • 插入操作：

      • insert 方法在树中插入新节点。
      • BSTInsert 方法将节点插入到二叉搜索树中，同时保持 BST 性质。
      • fixViolation 方法在插入后修复红黑树的性质

    • 旋转操作：

      rotateLeft 和 rotateRight 方法用于旋转操作，保持红黑树的平衡

    • 遍历操作：

      • inorder 方法进行中序遍历，打印节点
      • levelOrder 方法进行层次遍历，打印节点

    5. 标准库中的 set 和 map

    • set
      • set 是一个排序后的容器，不允许重复，底层基于红黑树实现
      • 虽然其插入删除的效率理论上是 O(log⁡N)O(\log{N})O(logN)，但可能需要进行旋转和重新着色等平衡操作，带来额外的开销。
      • 其内存分配也是动态的，插入新元素时需要重新分配内存。而相比于 vector 来说，vector 只在必要时进行内存分配
      • set 缓存效率较差，因为它在内存中不是连续存储的，导致缓存不易命中
    • map
      • map 用来存储排序后由键值组成项的集合，键必须唯一，底层基于红黑树实现

    3. 哈希表

    哈希表是一种记录键值关系的表结构。键值的映射关系称之为哈希函数。

    当两个键映射到同一个值时称之为哈希冲突。

    std::unordered_set 和 std::unordered_map 都基于哈希表（hash table）实现

    1. 哈希冲突

    • 分离链接法

      将映射到同一个值的所有元素保存到一个链表中。使用哈希函数确定某个链表，然后再对这个链表进行查询。

    • 再哈希法

      当冲突发生时尝试选择另外一个单元，为此还需要另一个哈希函数进行再哈希。

    4. 优先队列

    • 优先队列是一种抽象数据类型，类似于普通队列，但每个元素都有一个与之关联的优先级
    • 在优先队列中，元素按照优先级顺序被访问或移除
    • 优先队列有很多种实现方法，比如使用一个链表，并一直使它保持有序状态。但最常用的方式是使用二叉堆

    1. 二叉堆

    堆是一种特殊的树形结构，常用于实现优先队列。在堆中，每个结点都有一个优先级，且满足以下两个性质

    • 堆序性质 (Heap Property)：对于最大堆 (或最小堆)，任意结点的值都大于等于 (或小于等于) 其子节点的值
    • 完全二叉树性质 (Complete Binary Tree Property)：堆是一棵完全二叉树，即除了最后一层，其他层的结点都是满的，最后一层的结点尽可能地靠左排列
      • 一棵高为 hhh 的完全二叉树有 2h2^h2h 到 2h+12^{h+1}2h+1 个结点。这意味着，完全二叉树的高是 ⌊log⁡N⌋\lfloor \log{N} \rfloor⌊logN⌋

    如果使用数组表示二叉堆，那么一个事实是：对于数组中任一位置 iii 上的元素，其左儿子在位置 2i2i2i 上，右儿子在 2i+12i + 12i+1 上，其父亲在 ⌊i2⌋\lfloor\frac{i}{2} \rfloor⌊2i​⌋ 上。

    下面是使用数组实现二叉堆（最小堆）的一个例程：

    • 插入

      如果要在堆中插如一个元素 X，那么先在末尾开辟内存并放到末尾以满足完全二叉树性质。如果不满足堆序性质，那么不断与其父结点比较并决策是否需要交换位置，直至满足堆序性质为止

    • 删除最小值

      如果要删除堆的最小值，即二叉堆的根结点，也就是数组的第一个元素。首先将末尾的元素直接赋值给第一个元素代表第一个元素被删除，并将 currentSize 减 1，这是为了满足完全二叉树性质。然后为了满足堆序性质，不断将第一个元素与其子节点比较并决策是否需要交换位置，直至满足堆序性质为止

      percolateDown 函数的作用是下滤

    • buildHeap

      从倒数第二层开始到根结点为止，从后往前每个结点都下滤

    5. 排序

    1. 插入排序

    插入排序的基本思想是：对于当前下标 p，其前面 (p-1) 个元素都是排好序的 (p >= 1)。然后将 p 插入到前面已牌序好的序列中。

    排序算法的稳定性：指的是在排序过程中，如果两个元素相等，排序后它们的相对顺序是否保持不变。

    插入排序的复杂度是 O(n2)O(n^2)O(n2)，稳定。

    2. 希尔排序

    希尔排序的核心思想是先将整个待排序的序列分割成若干子序列，然后分别对各子序列进行插入排序，从而使得整个序列基本有序，最后再对全体记录进行一次直接插入排序。

    步骤：

    • 选择增量序列（一般选择一个逐步减小的序列），常用的增量序列：dk=⌊n2k⌋d_k = \lfloor \frac{n}{2^k} \rfloordk​=⌊2kn​⌋

      其中 n 代表待排序数组的大小（如 n = 13 时，序列为：{6, 3, 1}​）

    • 对于序列中的每个增量，都进行一次分组的插入排序，使数组基本有序

    • 直到增量为1时，此时就相当于一次插入排序，此时数组已经基本有序，所以需要交换的数量可能大为减少

    最坏时间复杂度为 O(n2)O(n^2)O(n2)，不稳定。

    3. 堆排序

    堆排序首先建立一个二叉堆，然后执行 nnn 次 deleteMin 操作，然后再将这些元素记录到结果数组。

    建立二叉堆需要 O(n)O(n)O(n) 的时间，每次 deleteMin 花费 O(log⁡n)O(\log{n})O(logn) 的时间，所以时间复杂度为 O(nlog⁡n)O(n\log{n})O(nlogn)。

    堆排序是一个不稳定的排序。

    percDown 是下滤的过程，从上到下调整结点的位置，通过选择较大的子结点进行交换，确保父节点总是大于等于其子结点。

    4. 冒泡排序

    步骤：

    • 从数组的第一个元素开始，依次比较相邻的两个元素
    • 如果前一个元素大于后一个元素（或小，取决于排序顺序），则交换它们的位置
    • 每次遍历数组，最大的元素都会移动到数组的末端
    • 对未排序部分重复上述步骤，直至整个数组有序

    冒泡排序复杂度是 O(n2)O(n^2)O(n2)，稳定。

    5. 选择排序

    选择排序的工作原理是将数组分为已排序和未排序两部分。每次从未排序部分中选择最小（或最大）元素，将其放到已排序部分的末尾。

    冒泡排序复杂度是 O(n2)O(n^2)O(n2)，不稳定。

    6. 归并排序

    归并排序的基本操作是合并两个已排序的表。它采用分治法，将一个数组递归地分成两个子数组，分别进行排序，然后合并这两个有序的子数组。

    归并排序时间复杂度是 O(nlog⁡n)O(n\log{n})O(nlogn)，稳定

    7. 快速排序

    快速通过分治法（Divide and Conquer）将一个序列分成较小的子序列，然后递归地排序子序列。

    算法步骤：

    • 选择基准点（Pivot）：
      从数组中选择一个元素作为基准点（pivot）。选择基准点的方法有多种，包括选择第一个元素、最后一个元素、中间元素或随机选择

    • 划分（Partitioning）：

      重新排列数组，使得所有比基准点小的元素放在基准点的左边，所有比基准点大的元素放在基准点的右边。这个过程称为划分

    • 递归排序（Recursive Sort）：

      对基准点左边的子数组和右边的子数组递归地应用上述步骤，直到子数组的大小为零或一（此时数组已经有序）

    平均复杂度为 O(nlog⁡n)O(n\log{n})O(nlogn)，不稳定。

    3. 拓扑排序

    拓扑排序（Topological Sorting）是指对有向无环图（DAG，Directed Acyclic Graph）进行排序，使得对于图中的每一条有向边 u→vu→vu→v，顶点 uuu 在排序中出现在顶点 vvv 之前。这种排序适用于描述依赖关系的场景，例如任务调度、编译器中的编译顺序等。

    • 性质：

      仅适用于有向无环图（DAG）。若有环，必不可能进行拓扑排序，假设回路上任意两点 uuu 和 vvv，uuu 既可以在 vvv z之前，也可以在 vvv 之后

      一个图可以有多个不同的拓扑排序

    • 常见的拓扑排序算法

      • Kahn算法（基于入度的算法）

        Kahn算法使用入度（即指向某个顶点的边的数量）来实现拓扑排序

        步骤：

        1. 计算每个顶点的入度
        2. 初始化一个队列，将所有入度为 0 的顶点入队
        3. 出队列中的顶点，将其添加到拓扑排序结果中
        4. 对于每个被取出的顶点，减少所有与之相邻的顶点的入度。如果某个相邻顶点的入度变为0，则将其入队
        5. 重复步骤3和4，直到队列为空
        6. 如果拓扑排序结果中的顶点数量不等于图中的顶点数量，说明图中存在环，无法进行拓扑排序

    4. 最短路径

    1. 问题划分

    • 单源最短路径

      单源最短路径问题是指从图中的一个起点顶点到图中所有其他顶点的最短路径问题。这个问题的目标是找到从一个指定顶点 s（源点）到图中所有其他顶点的最短路径。

    • 多源最短路径

      多源最短路径问题是指找到图中每一对顶点之间的最短路径。也就是说，对于图中的任意两个顶点 uuu 和 vvv，求解从 uuu 到 vvv 的最短路径。

    • 负环

      负环是指图中的一个环，其所有边权之和为负数。负环的存在会导致最短路径问题变得复杂，因为在一个包含负环的图中，可以通过多次绕行负环来使路径长度无限减小。因此，负环使得图中无法定义最短路径。

    5. A* 算法

    算法思想：

    启发式搜索。（相当于 BFS + 贪心）

    A* 算法通过维护一个优先队列来存储当前已发现但未处理的节点，每次从中选择一个估价函数值最小的节点进行扩展。估价函数 f(n)f(n)f(n) 通常定义为：

    f(n)=g(n)+h(n)f(n) = g(n) + h(n) f(n)=g(n)+h(n)

    • g(n)g(n)g(n)：从起点到结点 nnn 的实际代价
    • h(n)h(n)h(n)：从结点 nnn 到终点的启发式估计代价

    启发式函数 h(n)h(n)h(n) 应该是一个低估，即它不会超过从节点 nnn 到终点的实际代价，以保证 A* 算法的最优性。

    • 曼哈顿距离（Manhattan Distance）

      公式：

      h(n)=∣x1−x2∣+∣y1−y2∣h(n)=∣x_1 - x_2| + |y_1 - y_2| h(n)=∣x1​−x2​∣+∣y1​−y2​∣

    • 欧几里得距离（Euclidean Distance）

      欧几里得距离适用于可以在任意方向上移动的情况，通常用于平面地图

      公式：

      h(n)=(x1−x2)2+(y1−y2)2h(n)= \sqrt{(x_1 - x_2)^2 + (y_1 - y_2)^2} h(n)=(x1​−x2​)2+(y1​−y2​)2​

      计算从当前节点 nnn 到目标节点的直线距离，允许在任意方向上移动

    • 切比雪夫距离（Chebyshev Distance）

      切比雪夫距离适用于可以在水平、垂直和对角线上移动的情况

      h(n)=max⁡(∣x1−x2∣,∣y1−y2∣)h(n) = \max{(|x_1 - x_2|, |y_1 - y_2|)} h(n)=max(∣x1​−x2​∣,∣y1​−y2​∣)

    1. 初始化：

      • 创建一个开放列表（Open List），存储待处理结点
      • 创建一个关闭列表（Closed List），存储已处理结点
      • 将起点加入开放列表，并设置其 ggg 值为 0，hhh 值为启发式估计值，fff 值为 g+hg + hg+h

    2. 循环处理：

      • 从开放列表中取出估价函数 fff 值最小的结点 nnn
      • 如果结点 nnn 是终点，则构建路径并返回
      • 否则，将结点 nnn 移到关闭列表中，并处理其所有邻居结点

      对于每个邻居结点 mmm：

      • 计算从起点到邻居结点 mmm 的新 ggg 值（即当前节点的 ggg 值加上从当前节点到邻居节点的边的权重）

      • 如果 mmm 在开放列表中但新 ggg 值较小，更新其 g、hg、hg、h 和 fff 值，并更新其父节点为当前节点

    4. 重复步骤 2 和 3，直到找到路径或开放列表为空（无路径可达）

    5. 最小生成树

    最小生成树（MST, Minimum Spanning Tree）是图论中的一个重要概念。它指的是在一个无向加权图中找到一个子图，使得该子图是一个树，并且包含所有的顶点，同时所有边的权重之和最小。

    概念：

    • 无向图：图中边是无方向的
    • 加权图：图中的每条边都有一个权重（或成本）
    • 生成树：包含图中所有顶点的一个子图，是一个无环连通图
    • 最小生成树：所有生成树中边权重之和最小的那个

    1. Prim算法

    Prim算法是一种逐步扩展的贪心算法，从一个顶点开始，逐步扩展到包含所有顶点

    过程：

    1. 初始化：从一个起始顶点开始，将该顶点标记为已访问
    2. 选择边：在已访问的顶点中，选择一条连接到未访问顶点的权重最小的边
    3. 扩展树：将选择的边和对应的未访问顶点加入生成树，并标记该顶点为已访问
    4. 重复步骤 2 和 3，直到所有的顶点都被访问过

    2. Kruskal 算法

    Kruskal算法是一种基于边的贪心算法，通过将图中的所有边按照权重从小到大排序，然后逐步添加边来构建最小生成树。

    过程：

    1. 初始化：将所有边按权重从小到大排序
    2. 选择边：从权重最小的边开始，逐条检查该边是否形成环
    3. 构建树：如果不形成环，则将该边加入生成树
    4. 重复步骤2和3，直到生成树包含所有顶点。

    7. 算法设计技巧

    1. 贪心

    贪心算法（Greedy Algorithm）是一种在每一步选择中都采取在当前状态下最优（即最具贪心）的选择的算法。其核心思想是，通过每一步都做出局部最优的选择，最终希望能得到全局最优解。贪心算法并不总是能找到最优解，但在某些特定问题上，它确实能找到最优解。

    基本要素：

    1. 选择性：每一步都做出当前看起来最好的选择
    2. 可行性：每一步的选择必须是可行的，即满足问题的约束条件
    3. 最优子结构：通过局部最优选择构建全局最优解的问题结构
    4. 贪心选择性质：整体问题的最优解可以通过一系列局部最优选择构成

    典例：

    • 活动选择问题

      给定一组活动，每个活动有一个开始时间和结束时间，选择尽可能多的互不重叠的活动

      步骤：

      1. 按照活动的结束时间从小到大排序
      2. 从第一个活动开始选择，选择与当前活动不重叠的下一个活动

    • 哈夫曼编码

      哈夫曼编码是一种广泛用于数据压缩的最优前缀编码算法。其核心思想是通过构建哈夫曼树，根据字符的频率分配变长的二进制编码，使得高频字符分配较短的编码，低频字符分配较长的编码，从而达到压缩数据的目的。

      算法步骤：

      1. 统计频率：统计每个字符在输入字符串中出现的频率
      2. 构建优先队列：将每个字符及其频率作为一个叶节点放入优先队列（小顶堆）。
      3. 构建哈夫曼树：
        1. 从优先队列中取出频率最小的两个节点，构建一个新的节点，其频率为这两个节点频率之和
        2. 将新节点放回优先队列
        3. 重复以上步骤，直到队列中只剩一个节点，这个节点就是哈夫曼树的根节点
      4. 生成哈夫曼编码：从根节点开始，向左分支赋值 “0”，向右分支赋值"1"，递归生成每个字符的哈夫曼编码

    2. 动态规划

    动态规划（Dynamic Programming, DP）是一种用于解决具有重叠子问题和最优子结构性质的问题的算法技术。其核心思想是将问题分解为子问题，通过存储子问题的解来避免重复计算，从而提高效率。动态规划通常用于求解最优化问题，如路径问题、序列问题和资源分配问题。

    • 重叠子问题：问题可以分解为子问题，且这些子问题会被重复计算

    • 最优子结构：一个问题的最优解可以通过其子问题的最优解来构建。即问题的整体最优解包含了子问题的最优解。

      子问题之间是独立的，解决某个子问题的最优解不会影响其他子问题的最优解

    基本步骤：

    1. 定义子问题：将原问题划分为相互重叠的子问题
    2. 确定状态：用一个或多个变量表示子问题的状态
    3. 状态转移方程：找到子问题之间的递推关系
    4. 初始条件和边界情况：确定问题的初始值和边界条件
    5. 计算顺序：根据递推关系和初始条件，按照某种顺序计算出每个子问题的解。

    实现方式：

    1. 自顶向下（备忘录法）：利用递归，从顶层问题开始，逐步解决子问题，并将结果存储起来，以便在后续计算中重用
    2. 自底向上（迭代法）：从最小的子问题开始，逐步计算出较大的子问题的解，直至得到原问题的解
