    Shadertoy Notes

    1. Intro

    1. Shader

    Shader（着色器）是运行在图形处理器（GPU）上的小程序，用于控制计算机图形中的各种视觉效果和数据处理。

    在图形渲染管线中，着色器通常分为多个阶段，每个阶段处理不同的图形数据。常见的着色器类型包括：

    1. 顶点着色器（Vertex Shader）

      • 作用：负责处理每个顶点的属性（如位置、法线、纹理坐标等）。它将顶点数据从对象空间转换到裁剪空间，并执行顶点级别的变换和计算（如视图变换、投影变换、骨骼动画等）
      • 输入：模型的顶点数据
      • 输出：变换后的顶点数据

    2. 片段着色器（Fragment Shader）

      • 作用：负责计算每个片段（像素）的颜色和其他属性。片段着色器通常是实现光照、纹理映射、透明效果、阴影等视觉效果的地方
      • 输入：从顶点着色器或插值器传递下来的数据，例如插值后的纹理坐标、颜色、法线等
      • 输出：最终写入屏幕（帧缓冲区）的像素颜色

    3. 几何着色器（Geometry Shader）

      • 作用：可以接收顶点着色器输出的一组顶点，生成额外的几何体（如三角形、点或线）。它允许动态创建或修改图元形状，常用于细分曲面、草地、粒子等效果
      • 输入：从顶点着色器输出的图元（如三角形、线段或点）
      • 输出：修改后的或新增的图元

    4. 计算着色器（Compute Shader）

      • 作用：独立于渲染管线之外，用于执行通用计算任务。这种着色器适合进行大量的并行计算，通常用于图像处理、物理模拟、粒子系统等
      • 输入/输出：可以是任意类型的数据，包括纹理、缓冲区等，结果可以是图像或其他形式的数据

    5. 曲面细分着色器 (Tessellation Shader)

      • 作用：曲面细分着色器是用于将简单的几何体细分为更复杂的图形。它的目标是提高几何体的细节层次，尤其是在近距离观察时，同时尽量减少性能开销。曲面细分着色器非常适合处理光滑曲面和生成逼真的高细节模型，比如角色的皮肤、衣物，或者复杂的自然景观

      • 工作流程：

        曲面细分着色器通常由两个阶段组成：

        细分控制着色器 (Tessellation Control Shader, TCS)：决定几何体如何被细分。可以基于距离、视角等动态地调整细分级别

        细分评估着色器 (Tessellation Evaluation Shader, TES)：接收经过细分的顶点并计算其在曲面上的位置，通常基于贝塞尔曲线或 B样条曲线等曲面表示法

        这两个阶段之间由一个称为 细分器 (Tessellator) 的固定功能单元处理，它负责将原始几何体细分成更小的三角形或四边形

    6. 光线追踪着色器 (Ray Tracing Shader)

      用于基于光线追踪技术的渲染，如在 Vulkan 和 DirectX Ray Tracing 中应用的光线生成、闭合和击中着色器

    7. 后处理着色器 (Post-processing Shader)

      通常作为片段着色器的一部分，用于对已经渲染的图像进行后处理效果，例如模糊、边缘检测、色彩校正等

    2. Shadertoy

    Shadertoy 利用 WebGL API 使用 GPU 在浏览器中渲染图形。但在 Shadertoy 中仅可使用片段着色器，渲染场景依赖于 ray marching 和 SDF 等计算代价高昂的方法。

    3. fragCoord

    在 ShaderToy 中，fragCoord 是片段着色器中的一个输入参数，表示当前片段的屏幕坐标。

    假设当前绘制屏幕的宽和高分别为 w\mathbf{w}w 和 h\mathbf{h}h，那么像素从左下角 (0, 0) 开始，到右上角 (w\mathbf{w}w, h\mathbf{h}h) 结束。但需要注意的是，左下角的初始坐标是 (0.5, 0.5) 而非 (0, 0)，因为图形 API 总是将像素坐标定义在像素中心。虽然在大多数情况下这并不会对于我们的绘制带来影响。
