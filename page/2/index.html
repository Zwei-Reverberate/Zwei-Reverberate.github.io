C++ Smart Pointer

2022-11-01

1. RAII 机制

Resource Acquisition Is Initialization (RAII) 的简称，是 C++ 中一种管理、避免泄露的惯用法。利用的就是 C++ 构造的对象最终会被销毁的原则。

RAII 的做法是使用一个对象，在其构造时获取对应的资源，在对象生命周期内控制对资源的访问，使之始终保持有效，最后在对象析构的时候，释放构造时获取的资源。这样做的好处是不用开发人员显式地去释放资源，避免因考虑不周而造成的资源泄露。

本篇笔记源自 C++ 并发编程实战

1. 预备知识

1. Main 函式

所有的 C++ 程序都必须包含一个 main 函式，否则编译器会报错。（动态链接库和静态库没有 mian 函式）。

main 函式是源代码开始执行的地方，但在进入 mian 函式子之前，所有没有显式初始化的静态类成员都被设置为0了。在 Microsoft C++ 中全局的静态对象也在进入 main 之前被初始化。

main 所受的限定：

  • 不能被重载
  • 不能被声明为内联函数
  • 不能被声明为静态函数
  • 无法获取其地址
  • 不能从其他地方被调用
