CMake 学习笔记

2023-09-03

1. CMake 简介

1. 工程构建

构建 (Build) 是程序开发的步骤之一。对于一个项目来说，其中可能包含不同的组件和模块，引用了各种第三方库，如果手动进行编译连接效率是比较低下的。

所以我们一般会使用工程构建的方式全自动完成项目的编译，链接和打包。

一些常见的 C++ 构建工具：GUN Make，CMake，MSBuild，Scons，Waf 等。

2. CMake

CMake 是一个开源的跨平台的 C++ 构建工具。

CMake 自身不带有编译工具，使用 CMake 需要使用其他的如 MSVC，GCC，Clang 等编译工具。

2. 基本使用

1. 环境配置

本笔记的编译工具使用 Visual Studio 的 MSVC，下载安装好 Visual Studio 即可。

下载好 CMake 直接安装即可，注意配置好环境变量，在控制台中输入 “cmake –version” 出现版本号视为安装成功。

3. CMake 变量

CMake 中变量是 ${变量名} 的形式。

如 project(general) 命令自动生成了 PROJECT_NAME 这个变量，PROJECT_NAME是变量名，${PROJECT_NAME}是变量值，值为 general。

又如 CMAKE_BINARY_DIR 是 CMake 的内置变量，代表 CMake 内部定义的构建目录，所有的二进制文件都在这个文件夹中产生。

我们可以使用 message 命令打印出我们需要的信息，例如可以输出我们的 build 目录：

CMake 常见的内置变量包括：

变量名 说明
CMAKE_SOURCE_DIR 源代码根目录，所有工程顶层目录
CMAKE_CURRENT_SOURCE_DIR 当前处理的 CMakeLists.txt 所在的路径
PROJECT_SOURCE_DIR 当前 project 顶层目录
PROJECT_BINARY_DIR 工程构建目录
