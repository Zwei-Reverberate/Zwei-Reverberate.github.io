1. 预备知识

1. Main 函式

所有的 C++ 程序都必须包含一个 main 函式，否则编译器会报错。（动态链接库和静态库没有 mian 函式）。

main 函式是源代码开始执行的地方，但在进入 mian 函式子之前，所有没有显式初始化的静态类成员都被设置为0了。在 Microsoft C++ 中全局的静态对象也在进入 main 之前被初始化。

main 所受的限定：

  • 不能被重载
  • 不能被声明为内联函数
  • 不能被声明为静态函数
  • 无法获取其地址
  • 不能从其他地方被调用

2. 并发

1. 概念

计算机系统中的并发：在单个系统里同时执行多个独立的活动。

  • 并发假象：通过快速任务切换看起来像是任务在并行发生。（任务切换会造成时间和空间上的额外开销）
  • 硬件并发：拥有多个处理器或多核处理器真正并行超过一个任务

可执行程序：磁盘下一个文件，windows 下，一个扩展名为 .exe 的文件

进程：程序的一次执行过程，是拥有资源的基本单位

线程：一个基本的 CPU 单元，是进程中的一个实体，是独立调度的基本单位

  • 每个进程都有一个主线程，且主线程唯一，并和进程的生命周期一致
  • 同一进程下的所有线程共享地址空间，使得多线程并发的开销远小于多进程，代价是编码时必须确保当每个线程访问时所看到的数据是一致的
  • 线程并不是越多越好，每个线程都需要一个独立的堆栈空间（1M），线程之间的切换要保存很多中间状态，会耗费本该程序运行的时间

使用并发的原因：分离和性能

4. 在运行时选择线程数量

C++ 标准库中对此有所帮助的特性是 std::thread::hardware_currency()，这个函数返回一个对于给定程序执行时能够真正并发运行的线程数量的提示。例如在多核系统上它可能是 CPU 的核心数量。它仅仅是一个提示，如果该信息不可用则函数可能会返回0。

5. 标识线程

线程具有标识符，其类型是 std::thread::id。如果一个 std::thread 对象关联了一个执行线程，那么调用成员函数 get_id() 就可以获得线程的标识符。如果没有关联执行线程，那么get_id() 返回一个默认构造的 std::thread::id 对象表示“没有线程“。

std::thread::id 类型的对象可以自由复制和比较。相等表示代表着同一个线程或者都没有关联执行线程。若不等则代表不同线程或者其中一个没有关联执行线程。

当前线程的标识符可以通过调用std::this_thread::get_id() 获得。

3. 在线程间共享数据

1. 问题

所有线程之间共享数据的问题，都是修改数据导致的。

如果所有的共享数据都是只读的，就没有问题，因为一个线程所读取的数据不受另一个线程是否正在读取相同的数据而影响。

不变量：对于特定的数据结构总是为真的语句。例如”此变量包含了列表中项目的数量。“这些不变量在更新中经常被打破，尤其是在数据结构比较复杂或是更新需要修改超过一个值时。

可能发生错误的诱因之一：竞争条件。（如一个线程正在删除链表的结点，另一个线程正在读取链表）

避免有问题的竞争条件：

  • 方法一：使用保护机制封装你的数据结构

    确保只有实际执行修改的线程能够在不变量损坏的地方看到中间数据

    从其他访问该数据结构线程的角度看，这种修改要么还没开始，要么已经完成

  • 方法二：修改数据结构的设计及其不变量，从而令修改作为一系列不可分割的变更来完成，每个修改均保持其不变量

    这通常成为无锁编程，但难以尽善尽美

  • 方法三：将对数据结构的更新作为一个事务来处理。

    所需的一系列数据修改和读取被存储在一个事务日志中，然后在单个步骤中进行提交。如果该提交因为数据结构已被另一个线程修改而无法进行，该事务将重新启动。这称为软件事务内存（STM），不过 C++ 中对 STM 没有对 STM 的直接支持。

2. 用互斥元保护共享数据

由 C++ 标准提供的保护共享数据的最基本机制是互斥元（mutex）。

如果可以将所有访问某数据结构的代码块标记为互斥的，就能达到保护共享数据的目的。若任何线程运行了其中之一，其他试图访问此数据结构的线程就必须一直等到第一个线程完成。这就使得其他线程不可能看到损坏的不变量。

这就是互斥元的同步原语所能得到的。在访问共享数据结构之前，锁定与该数据相关的互斥元，当访问数据结构完成后，解锁该互斥元。线程库会确保一旦某个线程已经锁定某个互斥元，所有其他试图锁定相同互斥元的线程必须等待，直到此互斥元被解锁。这样就能确保所有线程看到共享数据自洽的一面，不带有任何损坏的不变量。

互斥元伴随的问题：死锁，保护过多火过少的数据。

在 C++ 中使用互斥元需要包含头文件 <mutex>，通过构造 std::mutex 的实例来创建互斥元，调用成员函数 lock() 来锁定它，调用成员函数 unlock() 来解锁它。不推荐直接调用成员函数，而是使用 std::lock_guard 类模板，它实现了互斥元的 RAII 惯用语法：在构造时锁定所给的互斥元，析构时解锁，从而保证互斥元始终被正确解锁。
