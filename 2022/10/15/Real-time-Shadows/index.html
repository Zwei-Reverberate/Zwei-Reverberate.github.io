1. 非常用的阴影计算方法

1. 投影阴影

  • 优势：高效，易于实现

  • 劣势：只适用于平坦表面

  • 计算方式

    假定场景中存在一个光源位于 (Xl, Yl, Zl), 一个需要渲染的物体，以及一个投影阴影的平面。

    直接使用一个变换矩阵将物体上的点 (Xw, Yw, Zw) 变换为相应阴影平面上的点 (Xs, O, Zs) 。

    然后直接将其生成的阴影多边形绘制出来，通常使用暗色和地面纹理的混合作为其纹理。

2. 阴影体

  • 优势：高度准确，不易产生伪影

  • 劣势：计算代价高昂

  • 计算方式

    先找到被物体阴影覆盖的阴影体，之后减少视锥体和阴影体相交部分中的多边形的颜色强度。

    几何着色器可以用于计算阴影体，模板缓冲区可以用于判断像素是否在阴影体内。

2. Shadow Mapping

1. 概述

  • 主流的实时阴影计算方式
  • 相对高效，受到多数硬件支持
  • 基本思想：光线不可视之处即为阴影
  • 计算方式：
    • 进行两轮 Pass，第一轮从光源位置渲染场景，第二轮从设定的摄像机渲染场景。（Pass 即渲染一遍场景）
    • 第一轮 Pass：(计算Shadow Map)
      • 从光源位置出发，并朝向光线方向渲染
      • 使用一张 Texture 记录每个像素的光源与最近的对象的距离/深度（和 Z-Buffer 类似）。这张 Texture 即为阴影贴图 Shadow Map
    • 第二轮 Pass：
      • 正常渲染场景流程不变
      • 对于每个像素，还要使用相应的变换矩阵将其变换到光源空间坐标 (x, y, z)，并查询第一轮 Pass 生成的 Shadow Map 中(x, y) 对应的存储的深度值 depth
      • 若 z > depth，说明该像素被遮挡，处于阴影中，就要对其作出相应的处理

2. Shadow Bias

如不加处理直接使用 Shadow Map 的话，有可能出现 Show acne（表现为黑白波浪纹覆盖在场景表面，亦称为自阴影）。

其产生原因在于舍入误差。因为阴影贴图受限于分辨率，在距离光源比较远的情况下，去 Shadow Map 中查找深度信息时计算的纹理坐标与实际坐标不一定能够完全匹配。

例如，设当前像素变换到光源空间的后的对应坐标为 (x1, y1, z1)，它的一个相邻像素变换后的坐标为 (x2, y2, z2)，而 Shadow Map 受限于分辨率对于 (x1, y1) 和 (x2, y2) 都返回了相同的深度 depth，而恰好 z1 > depth 被判断在阴影之中，而 z2 < depth 被判断不在阴影之中，自阴影问题也因此产生。而正确的情况应该是 (x1, x2) 本该对应的 depth’ > z1 > depth。

为了解决这个问题，可以队当前表面深度应用一个阴影偏移，相当于将物体向光源方向移动了一下，用上面的例子来说就是实质上减小了 z1 使其能够小于 depth。通常一个很小的偏移就可以补偿各类舍入误差。

产生原因：bias 设置不当，物体偏移太大，使物体看起来像是浮起来了一样。

解决方法：在渲染 Depth Map 的时候使用正面剔除 (front face culling)，这样可以使用几何体的背面作深度记录。

3. Percentage Closer Filtering (PCF)

1. 阴影锯齿

Shadow Mapping 所使用的 Texture 有一个固定的分辨率，易产生多个像素对应同一个纹理像素。这样一来多个像素就会从 Shadow Map 的同一个深度值进行采样，这几个像素得到的就是相同的结果，由此产生了锯齿。

解决的方式可以是增加深度贴图的分辨率，或者尽可能让光的视锥接近场景。

2. 软阴影

使用软阴影也是解决阴影锯齿的方案。

真实世界出现的阴影大多数都是软阴影，边缘会发生不同程度的模糊。原因是真实世界的光源很少是点光源，而且物体自身的材质会产生一些反射。

软阴影的特性：

  • 距离物体越远的阴影越柔和，越近则越锐利
  • 距离物体越近颜色越暗
  • 半影（Penumbra）。一些阴影区域的光被遮挡一部分，但并非完全遮挡，是阴影过渡的区域

3. PCF

PCF 是多种生成软阴影方法中的其中一种。

  • 算法思想：

    计算阴影时不仅考虑单个的采样点，而是在一定范围内进行多重采样，减少阴影的锐利程度，所以得到的值不是非0即1，而是0到1中间的值。这也符合真实情况中的半影。

  • 注意点：

    • 不是对最后已有锯齿的阴影进行 Filter
    • 亦不是对 Shadow Map 进行 Filter
    • 而是找到目标像素对应纹素周围的纹素，得到的是该像素是否处于阴影中0或1的平均，平均下来是一个在0到1之间的值，而非周围像素深度的平均

  • 找周围像素这个行为如何实现？

    定义一个采样半径 R，可以以此就近采样，可以选择不同的分布采样函数。

4. Percentage Closer Soft Shadows (PCSS)

上文已述及，软阴影应随着与物体距离的增加而越模糊，而 PCF 生成的软阴影并不能产生这样的效果，因为其采样半径 R 是固定的，而 PCSS 则是动态修改采样半径的大小，以得到软硬程度不同的阴影。

PCSS 通过半影的大小估计每次的采样半径 R。

其中 WLight 是光源面尺寸，dBlocker 是遮挡物的深度，dReceiver 是被投物(shading point) 的深度。

而 dBlocker 需要单独计算，而不能直接用单个坐标采样 Shadow Map 的深度直接作为 dBlocker ，其原因在于这种做法等同于单一直线连向光源面中心，若未碰到遮挡物（即 dBlocker = dReceiver）则半影直接为0，失去了计算的意义。

解决此问题可以对 Shadow Map 进行一定范围内的采样，若得到的深度小于 dReceiver 则认为碰到了遮挡物，并将其算入平均遮挡深度，采样后的平均深度就可以作为 dBlocker。

算法过程：

  1. Blocker search

    在某个区域内计算平均遮挡深度d

  2. Penumbra estimation

    通过计算出来的平均遮挡深度来计算半影的大小

  3. Percentage Closer Filtering

    根据半影动态确定不同区域的采样范围，然后以此计算 PCF
