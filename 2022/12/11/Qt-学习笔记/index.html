Qt 学习笔记

2022-12-11

1. Qt 简介

Qt 是一套跨平台的应用程序开发类库。

Qt 经常被当作 GUI 库来使用，但它的内容并不止于此，它还包含了网络编程，图形编程，音视频编程，多线程编程，文件操作等丰富的内容。

2. 环境配置

这里配置的环境是 Visual Studio 2022 和 Qt 6.2.4 版本。请选择适合需要的版本。此文不涉及 Visual Studio 的配置。

首先进入 Qt 的下载地址 选择期望的版本和对应操作系统的下载安装器，这里选择的是 4.4 目录下的 qt-unified-windows-x64-4.4.0-online.exe，注册并登录 Qt 账号，设定安装路径，并选择 Custom installation。选择 Qt 5.2.4 目录下的 MSVC 2019 64-bit 和 MinGW 安装即可。

简单介绍一下两种不同的 C++ 编译器 MSVC 和 MinGW：

  • MSVC (Microsoft Visual C++ Compiler) 是微软公司在 Windows 下的编译器，也是 Visual Studio 默认使用的编译器
  • MinGW (Minimalist GNU for Windows) 是 GUN 在 Linux 上的 gcc/g++ 移植到 Windows 上是产物

在编译 Qt 程序时可以选择适合需要的编译器。

安装完成后在 Qt Creator 中创建一个 Qt Widgets Application 程序能正常运行视为安装成功。

我们希望在 VS 中使用 Qt，所以在 VS 中安装扩展 Qt Visual Studio Tools。安装完成后，在 Qt VS Tool 扩展中设定 Qt Version，这里设定为 msvc2019_64，当然也可以按需要设定为 MinGW。设定完成后同样创建一个Qt Widgets Application 程序验证是否安装成功。

到此为止环境配置完成，可以进行后续的学习和开发了。

4. 信号和槽机制

  • 信号：特定情况下触发的事件

    如 PushButton 最常见的信号就是鼠标点击发射的 clicked() 信号

    GUI 程序设计的主要内容就是对界面上各组件的信号进行响应，只需要知道什么情况下发射哪些信号，合理地去响应和处理这些信号就可以了

  • 槽：对信号响应的函数

    槽是一个函数，具有一般函数所有的功能。

    与一般函数的区别在于，槽函数可以与一个信号关联，当信号被发射时，关联的槽函数被自动执行。

其中, sender 是发射信号对象的名称，signal() 是信号名称。信号可以看做是特殊的函数，需要带括号，有参数时还需要指明参数。receiver 是接收信号的对象名称，slot 是槽函数的名称，需要带括号，有参数时需要指明参数。

其作用就是将 btnClose 按钮的 clicked() 信号与窗体 (Widget) 的槽函数 close() 关联，这样，当单击 btnClose 按钮时，就会执行 Widget 的 close() 槽函数。

信号和槽的规则：

  1. 一个信号可以连接多个槽

    当一个信号与多个槽函数关联时，槽函数按照建立连接时的顺序依次执行

  2. 多个信号可以连接同一个槽

  3. 一个信号可以连接另外一个信号

  4. 严格的情况下，信号与槽的参数个数和类型需要一致

    至少信号的参数不能少于槽的参数，如果不匹配，就会出错

  5. 在使用信号与槽的类中，必须在类的定义中加入宏 Q_OBJECT

  6. 当一个信号被发射时，与其关联的槽函数通常被立即执行，就像正常调用一个函数一样。只有当信号关联的所有槽函数执行完毕后，才会执行发射信号处后面的代码
